谷歌云平台业务介绍

缘起#

2021.05.19

公司在使用谷歌云平台(google cloud platform)，这篇用于了解谷歌云平台的能力，知道谷歌在解决什么问题。以下内容均来自谷歌公开的文档。

学习链接#

  • Google Cloud Blog 谷歌云的博客，上面会分享一些有份量的文章帮助理解
  • Google Cloud Infrastructure 以可视化的方式理解谷歌云的运转流程，确实非常酷炫
  • Google Cloud Infrastructure 这里细窥谷歌云架构的设计

谷歌云的概念#

这里是一些谷歌云定义的概念，了解谷歌云就要先理清这些概念。

这部分可以只记标题，下面文章提到的时候再回来查询。

数据中心(data center)#

Google 在美洲、欧洲和亚洲都设有数据中心，将其用于计算和存储。

数据中心是 Google 内容和服务的核心。Google 建立了一个大型的专用数据网络，将其所有数据中心链接在一起，以便可以在多个站点之间复制内容以增强弹性，并可以最接近最终用户的方式交付服务。

简单理解#

谷歌的数据中心应该和国内的互联网数据中心是一个概念，都是负责基础物理设备的机房，其关注点在于提供稳定的基础服务。

而谷歌和苹果类似，也在关注可再生能源和碳排放，从而为环境做出贡献。

对比下中国资本和国际资本

  • 国际资本起了个好的榜样。因为我们普遍认为资本只重视利益，缺乏社会责任感。谷歌和苹果这么做：
    • 既可以美化自身形象，招聘到有理想、有能力的高级人才为其打工；
    • 又能为人类的可持续发展做出贡献，让别人在指责资本作恶时抓不住这方面的把柄，而且也是自身话语权的有力武器。

边缘网络#

网络边缘位置#

  • 网络边缘位置

网络边缘位置，英文 network edge locations。

简单理解#

谷歌的网络边缘位置和边缘计算没有什么关系。人类社会聚集在都市圈，通常云中心和其网络都会建立在都市或者都市附近。所以谷歌云在这些都市建立 100 多个网络边缘位置，这些网络边缘位置连接谷歌云会很快速，例如你要部署代码或者在云控制台页面修改配置，就会很方便。

专用互连#

专用互连可在本地网络和 Google 网络之间提供直接物理连接。通过专用互连，可以在网络之间传输大量数据，这比在公共互联网上购买额外带宽更加经济实惠。

简单理解#

不从公网绕远，而是直接通过运营商和谷歌的专线传输数据。

基础架构现代化改造#

对您的云基础架构进行现代化改造，并在 Google Cloud 上以原生方式运行关键工作负载（例如 VMware、SAP、Oracle 和 Windows 工作负载）。

数据库#

借助具备安全性、可靠性和高可用性的全代管式数据服务，迁移和管理企业数据。

应用现代化改造#

对旧式应用进行现代化改造，并构建可利用 Kubernetes、容器和其他云原生技术的新服务。

智能分析#

利用无服务器的全代管式分析平台，告别复杂的分析结果，受益于富有实用价值的分析洞见。

AI 和机器学习#

让您的团队掌握 AI 和机器学习解决方案，以提升客服和招聘等部门的效率。

安全#

借助成熟的安全解决方案检测、调查和应对在线安全威胁，以帮助保护您的业务。

业务应用平台#

借助 API 扩展您的现有数据，从而以安全的方式自动执行流程并轻松创建应用，而且无需编写代码。

工作效率与协作#

借助 Google Workspace，以更快速、更智能、更具协作性的方式开展工作。Google Workspace 是一种工作场所高效办公解决方案，包含 Gmail、Google 文档、云端硬盘和 Meet 等应用。

零售#

在从店铺运营到商品展示再到客户获取与留存的整个零售价值链中，保持敏捷性。

金融服务#

在快速变化的金融市场中管理风险，更及时地做出决策并保持竞争优势，同时确保合规性。

医疗保健与生命科学#

提供个性化的患者体验、实现研发现代化，并解决医疗保健与生命科学领域中的棘手难题。

媒体与娱乐#

创作世界级内容、简化工作流、快速推出新型数字服务，带来全新的受众体验。

公共部门#

帮助改善便民服务、提高运营效率，并将经过实践检验的创新成果运用于政府机构。

游戏#

凭借无需运维的基础架构和实时数据洞见，吸引全球玩家群体并助力您的游戏开发人员。

制造#

跨制造价值链助力您的工作人员并推动收入增长、实现卓越运营和创新。
