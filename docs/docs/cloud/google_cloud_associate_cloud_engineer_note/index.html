Associate Cloud Engineer Exam 资料#

问题 Associate Cloud Engineer(ACE) 是什么？#

助理云工程师(Associate Cloud Engineer)负责部署应用程序、监控运营和管理企业解决方案。此人能够使用 Google Cloud Console 和命令行界面来执行基于平台的常见任务，以维护一个或多个部署的解决方案，这些解决方案利用 Google Cloud 上的 Google 管理或自我管理服务。

问题 Google Associate Cloud Engineer Exam 是什么？#

Associate Cloud Engineer Exam(助理云工程师考试) 是 Google Cloud 认证的能力考试，可以检验此人专业知识的掌握情况，通过助理云工程师考试后，表示此人有能力使用 Google Cloud 技术实现企业转型。

Associate Cloud Engineer Exam 评估以下方面的能力：

  • 设置云解决方案环境
  • 规划和配置云解决方案
  • 部署和实施云解决方案

关于认证考试：

  • 时长：2 小时
  • 考试费用：$125（加税）
  • 语言：英语、日语、西班牙语。
  • 考试形式：多项选择和多项选择，远程或亲自在考试中心进行。
  • 考试交付方式：
    • 从远程位置参加在线监考考试，查看在线考试要求。
    • 在考试中心参加现场监考的考试，找到你附近的考试中心。
  • 推荐经验：6 个月以上 Google Cloud 实践经验。

  • 云存储桶中有一个对象，与一家外部公司共享。

问题思路

  • 要求安全，不能泄漏数据。所以 URL 的 query 参数中有签名信息。如果有人去掉 query 参数访问路径，拿不到资源

  • 你为每个开发人员提供了一个单独的 Google Cloud 项目，他们可以将其作为个人沙盒来试验不同的 Google Cloud 解决方案。
  • 如果有任何开发人员在他们的沙盒环境中每月花费超过 500 美元，你希望得到通知。

解题思路：

每人一个项目，然后一个项目一个预算提醒

  • 为每个项目创建一个预算，并对所有这些预算配置预算警报。

问题 158#

note
  • IAM
  • 公司在一个项目中集中使用了大量的 Google Cloud 服务。所有团队都有特定的项目用于测试和开发。
  • DevOps 团队需要访问所有的生产服务，以便执行他们的工作。
  • 防止 Google Cloud 产品的变化在未来扩大他们的权限。

解题思路：

  • 创建一个结合所需权限的自定义角色。授予 DevOps 团队在生产项目上的自定义角色。

问题 159#

note
  • Cloud Storage lifecycle
  • Cloud Storage signed URL

你正在建立一个应用程序，处理从成千上万的供应商上传的数据文件。你的应用程序的主要目标是数据安全和让超时的数据过期。你需要设计这个应用程序。

  • 限制访问，使供应商只能访问他们自己的数据
  • 让供应商对数据的写入权限只有 30 分钟。
  • 删除超过 45 天的数据。

你有一个非常短的开发周期，你需要确保该应用程序需要最小的维护。你应该使用哪两种策略？

选择两个。

  • A. 建立一个生命周期策略，在 45 天后删除 Cloud Storage 对象。
  • B. 使用签名的 URL，允许供应商在有限的时间内访问存储他们的对象。

正确答案： AB

  • Google Cloud 中的多个应用程序需要连接到企业内部的数据库服务器，你想避免在数据库的 IP 发生变化时，不得不改变所有应用程序的 IP 配置。你应该怎么做？

Cloud Run 是按照使用时间计费

问题 165#

note
  • IAM
  • Service Account
  • 在 BigQuery 上运行一个数据仓库。
  • 一个合作伙伴公司正在提供一个基于你的数据仓库中的数据的推荐引擎。
  • 该合作伙伴公司也在 Google Cloud 上运行他们的应用程序。
  • 他们在自己的项目中管理资源，但他们需要访问你项目中的 BigQuery 数据集。
  • 预期：为伙伴公司提供对数据集的访问。

解题思路：

合作伙伴公司的账号和自己公司的账号不在一起管理，所以才从自己公司给合作伙伴公司单独开通一个服务账户。配置可以访问 BigQuery 数据集。

  • 要求合作伙伴在他们的项目中创建一个服务账户，并让他们的服务账户访问你项目中的 BigQuery 数据集。

解题思路：

  • 用新版本的应用程序创建一个新的修订。在这个版本和当前运行的版本之间分流。

问题 167#

  • 公司开发了一款手机游戏，并部署在 Google Cloud 上。
  • 游戏玩家通过互联网用他们的个人手机连接到游戏。
  • 当玩家在多人游戏模式下进行游戏时，游戏会发送 UDP 数据包来更新服务器上的玩家的行为。
  • 你的游戏后端可以在多个虚拟机（VM）上扩展，
  • 想通过一个IP地址暴露这些虚拟机。

解题思路：

  • 在应用 server 前配置一个外部网络负载平衡器。
  • 内部负载均衡用于 VPC 内部进行负载，只有内网 IP，没有公网 IP
  • 外部负载均衡可以有一个公网 IP

问题 168#

note
  • Cloud Storage
  • cron job
  • 在为一家医院工作，该医院将其医疗图像存储在一个内部数据室中。
  • 该医院希望使用 Cloud Storage 对这些图像进行存档存储。
  • 医院希望有一个自动化的过程来上传任何新的医疗图像到 Cloud Storage。
  • 设计和实施一个解决方案。

解题思路：

  • 创建一个脚本，使用 gsutil 命令行界面来同步企业内部存储和 Cloud Storage。将该脚本安排为一个 cron job。

问题 169#

note
  • Cloud Storage
  • IAM
  • Audit Logs - Data Access Log
  • 审计员想查看组织在 Google Cloud 中的数据使用情况。
  • 审计员最感兴趣的是审计谁访问了 Cloud Storage bucket 中的数据。
  • 帮助审计员访问他们需要的数据。

解题思路：

  1. 默认情况下，数据访问审核日志处于停用状态，因为这些审核日志可能非常大。如果您想针对 BigQuery 以外的 Google Cloud 服务写入数据访问审核日志，则必须明确启用这些日志。启用这些日志可能导致我们因额外的日志使用量而对您的 Cloud 项目收取费用。
  2. 如需查看这些日志，必须具有以下 IAM 角色：Logging/Private Logs Viewer 或 Project/Owner。

所以需要开启 Data Access Log

  • 为他们想要审计的 bucket 打开 Data Access Log(数据访问日志)，然后在日志查看器中建立一个查询，对 Cloud Storage 进行过滤。

激活服务帐号 gcloud auth activate-service-account [ACCOUNT] --key-file=[KEY_FILE]

  • 使用命令 gcloud auth activate-service-account 并将其指向私钥。

问题 179？？？#

note
  • VPC
  • firewall rule
  • 公司正在将其全部工作负载转移到 Compute Engine。
  • 一些服务器应该可以通过互联网访问，而其他服务器应该只能通过内部网络访问。
  • 所有的服务器都需要能够通过特定的端口和协议相互交谈。
  • 当前的本地网络依赖于公共 server 的 demilitarized zone (DMZ) 和私有 server 的 Local Area Network (LAN)。
  • 需要设计 Google Cloud 上的网络基础设施来满足这些要求。

解题思路：

    1. 创建一个单一的 VPC，为 DMZ 建立一个子网，为 LAN 建立一个子网。2. 设置防火墙规则，开放 DMZ 和 LAN 子网之间的相关流量，并设置另一个防火墙规则，以允许 DMZ 的公共入口流量。
