Classless Inter-Domain Routing (CIDR)#

What Is CIDR

CIDR 代表无类域间路由，是一种改进 IP 地址分配的 IP 寻址方案。

它取代了基于 A、B 和 C 类的旧系统。该方案还有助于大大延长 IPv4 的寿命并减缓路由表的增长。

Cloud Load Balancing#

  • 全球范围的自动扩缩和负载平衡。可以使用智能自动扩缩功能来增加或减少资源。
  • 通过单个任播 IP 实现全球负载平衡。可以将一个任播 IP 用作全球各区域中所有后端实例的前端。
  • 软件定义的负载平衡。适用于所有流量的完全分布式、软件定义的代管式服务。
  • 每秒钟超百万次查询。构建于 Google 所用的前端服务基础架构之上。支持每秒处理超过一百万次查询，并且保证一致的高性能和低延迟。
  • 无缝自动扩缩。随着用户和流量的增长而调整（包括将流量转移到全球其他可以接收流量的区域），轻松处理数量巨大的瞬间意外峰值。自动扩缩功能无需预热，可以在短短几秒的时间里从零调节至所需的容量。
  • 内部负载平衡。可以为内部客户端实例构建可伸缩的高可用性内部服务，而不需要将负载平衡器暴露给互联网。
  • 支持领先协议。它不但支持在连接到后端时通过 HTTP/2 使用 gRPC 协议进行通信，而且还是第一个支持 QUIC 协议的大型公有云。

Cloud VPN#

Cloud VPN 通过 IPsec VPN 连接，将你的对等网络安全地连接到 Virtual Private Cloud (VPC) 网络。两个网络之间传输的流量由一个 VPN 网关加密，再由另一个 VPN 网关解密。此操作可以保护你的数据在互联网上传输时的安全。你还可以将两个 Cloud VPN 实例相互连接。

instance group(实例组)#

实例组是可以作为单个实体进行管理的虚拟机实例的集合。

两种虚拟机实例组：

  • 托管实例组：可以在多个相同的虚拟机上运行应用。
    • 可以利用自动化服务让工作负载具有可扩缩性和高可用性，这些服务包括自动扩缩、自动修复、区域（多地区）部署和自动更新。
  • 非托管实例组：非托管实例组可以包含异构实例，这类实例可以在该实例组中任意添加和移除。
    • 非托管实例组不提供自动扩缩和自动修复功能
    • 不支持滚动更新和多区域
    • 不允许使用实例模板。
    • 如果需要对异构实例组应用负载平衡，或者需要自行管理实例，可以使用非代管实例组。

instance life cycle(虚拟机实例生命周期)#

虚拟机可能处于以下任一状态：

  • PROVISIONING：为虚拟机分配的资源。虚拟机尚未运行。
  • STAGING：获取资源，虚拟机正在准备首次启动。
  • RUNNING - 虚拟机正在启动或运行。
  • STOPPING：虚拟机正在停止。你请求停止，或者出现了故障。这是虚拟机进入 TERMINATED 状态后的临时状态。
  • REPAIRING：正在修复虚拟机。当虚拟机遇到内部错误或底层机器由于维护而不可用时，会进行修复。在此期间，虚拟机无法使用。如果修复成功，则虚拟机会返回上述状态之一。
  • TERMINATED：虚拟机已停止。你已停止虚拟机，或者虚拟机发生故障。你可以重启或删除虚拟机。
  • SUSPENDING：虚拟机正在暂停。你暂停了虚拟机。
  • SUSPENDED：虚拟机处于暂停状态。你可以恢复虚拟机或删除虚拟机。

在计算机科学以及其它领域中，最小权限原则是要求计算环境中的特定抽象层的每个模块如进程、用户或者计算机程序只能访问当下所必需的信息或者资源。 它赋予每一个合法动作最小的权限，就是为了保护数据以及功能避免受到错误或者恶意行为的破坏。最小权限原则也称为最少权限原则。

Virtual Private Cloud (VPC)#

VPC 网络概览#

适用于资源的代管式网络功能。

  • 灵活扩缩以及控制工作负载在区域范围及全球范围内如何进行连接
  • 无需在每个区域内复制连接或管理政策，即可访问 VPC
  • 在跨所有区域的 Google 网络基础架构上使用自己的 IP 地址

特点：

  • VPC 网络（包括其关联的路由和防火墙规则）属于全球性资源。它们与任何特定地区或区域均无关联。
  • 子网属于地区性资源。

VPC 网络对等互连概览#

  • VPC 网络对等互连允许跨两个 Virtual Private Cloud (VPC) 网络的内部 IP 地址连接（无论它们是否属于同一个项目或组织）。
  • VPC 网络对等互连可以连接 VPC 网络，以便不同 VPC 网络中的工作负载可在内部进行通信。流量保留在 Google 的网络中，不会通过公共互联网。
