配置解析#

基于项目根路径解析的绝对路径#

我们把 /@ 解析为项目根目录下的 src 目录。

+ import path from 'path'export default defineConfig({+   resolve: {+     alias: {+       '/@': path.resolve(__dirname, 'src')+     },+   },})

也就是说可以在项目中通过 /@/main.js 引用 src/main.js。

可以解决以下一些场景：

  1. a 文件引用 b 文件。在 a 文件所在的目录结构发生变化时（例如挪到其他目录下面），不必修改引用 b 文件的路径。
  2. a 文件引用 b 文件，但 a 文件处于一个较深层级的目录（a 位于 /1/2/3/4/5/6/a），b 文件处于一个较浅层级的目录（例如 /b），。
    • 如果是相对路径，a 需要通过 ../../../../../../b 来引用 b。
    • 如果是绝对路径，a 直接通过 /@/b 来引用 b。
