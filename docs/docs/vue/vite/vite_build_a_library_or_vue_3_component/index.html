使用 Vite 构建一个 library 或 Vue 3 组件

基本配置#

请参考 使用 Vite 为 Vue 3 应用程序配置开发环境和生产环境。

构建选项#

在 vite.config.js 中配置 library 的构建选项#

通过 build.lib 可以配置：

  1. entry：构建入口。
  2. fileName：输出文件名称。
  3. formats：输出遵循的模块规范，数据结构是字符串数组。
    • esm 表示遵循 ES modules 规范。
+ import path from 'path'export default defineConfig({  build: {+    lib: {+      entry: path.resolve(__dirname, 'lib/index.js'),+      fileName: 'index',+      formats: ['esm'],+    },})

在 vite.config.js 中配置不参与构建的 library#

通过 build.rollupOptions 配置想要排除的 library。

export default defineConfig({+  build: {+    rollupOptions: {+      external: [+        '@ant-design/icons-vue',+        'axios',+        'ant-design-vue',+        'nprogress',+        'vue',+        'vue-router',+      ],+    }+  },})

发布选项#

在 package.json 中配置引用模块的方式#

main 和 module 这两个字段，用于交给 webpack, Vite 等构建工具使用。可以参考 package.json 的 browser, module, main 字段优先级 这篇文章。

{  "main": "dist/index.umd.js",  "module": "dist/index.esm.js"}

在 package.json 中配置发布模块#

  1. files 放置要发布的目录和文件。
    • "lib"：如果不想暴露源码，记得不写源码目录。
    • "dist"：构建结果目录，记得将 module 和 main 指向到这里的文件。
  2. publishConfig 放置一些发布的配置
  3. publishConfig.registry 表示 npm registry 的基础 URL。
{  "files": [    "lib",    "dist"  ],  "publishConfig": {    "registry": "https://registry.npmjs.org/"  }}
