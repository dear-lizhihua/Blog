虚拟滚动列表

预备知识#

什么是真实滚动？#

想要了解"虚拟滚动"这个概念，那就要先了解对应的"真实滚动"这个概念。

假想我们有个一个外层容器 container 元素，还有一个内层列表 list 元素，列表内部是 1000 个 item 元素。

<div class="container">  <div class="list">    <div class="item"></div><!-- 1000 个 -->    ...  </div></div>

定义一种特殊情况：

  1. container 高度是 100px。
  2. item 每个高度是 10px，所有 item 总高度是 1000*10=10000px。
  3. list 高度跟随其内部 item 高度。

为什么要优化真实滚动？#

试想，container 高度 100px，item 每个高度是 10px。分析可知：

  • 为 1000 个 item 渲染了 1000 个 DOM 节点。
  • 但最多只能同时看到 (100px/10px)+1=11 个 DOM 节点。
    • 为什么要 +1，试想第 1 个节点滚动出去半个节点，第 11 个节点也会滚动进来半个节点。
    • 但不会超过 12 个，因为第 1 个节点完全滚动出去，第 11 个节点也会完全滚动进来。
  • 有 990 个 DOM 节点不在可视区域内。

那问题来了，我们有必要为看到可见的 11 个 item，创建 1000 个 DOM 吗？

要知道通过 JavaScript 创建和操作 DOM，需要消耗计算和占用内存，远远超过创建和操作 JavaScript 对象。而且在 JavaScript 和 DOM 之间的交互上面是极为耗费计算的。

参考：前端为什么操作 DOM 是最耗性能的呢？ - justjavac的回答

什么是虚拟滚动？#

如果我们始终在滚动时只创建 11 个 DOM，并且一直修改这 11 个 DOM。就可以实现性能的大幅度提升：

  1. 避免通过 JavaScript 创建多余的 990 个 DOM，大量减少 JavaScript 和 DOM 之间的交互，减少了初始的计算量。
  2. 避免浪费开辟内存存放多余的 990 个 DOM，减少了创建无用内存。

虚拟滚动技术，本质上就是创建出一个 pool(池)，复用池内存储的 DOM 节点。

设计方案，理解虚拟滚动#

设计结构#

根据刚才的描述，我们此时重新设计结构，list 内部只有 11 个 item，我们给这些 DOM 编上号。

<div class="container">  <div class="list">    <div class="item">1</div>    <div class="item">2</div>    <div class="item">3</div>    <div class="item">4</div>    <div class="item">5</div>    <div class="item">6</div>    <div class="item">7</div>    <div class="item">8</div>    <div class="item">9</div>    <div class="item">10</div>    <div class="item">11</div>  </div></div>

设计样式#

为了便于理解，我们在 container 上面添加如下样式。

<style>*{box-sizing: border-box}body{font-size: 8px;}.container {overflow-y: auto;height:100px;border: 1px solid #f00;}.list{min-height: 10000px;}.item {height:10px;border: 1px solid #ccc;border-bottom: 1px solid #00f;}</style>

这个案例的样式设计说明如下：

  1. 为了便于取整计算，container 高度取 100px。
  2. 为了便于取整计算，每个 item 高度取 10px，而不取 20px。
  3. 为了设置滚动，container 添加 overflow-y: auto。
  4. 为了设置滚动，list 添加最小高度 10000px。
  5. 为了防止超出，每个 item 盒模型改为 border-box。
  6. 因为行高只有 10px，所以每个文字就展示 8px。
  7. 容器边框线是红色。
  8. 每个格子上 左 右边框线是灰色，下边框线是蓝色。

第一个步骤：列表未开始滚动#

第二个步骤：列表开始滚动，第 1 个节点滚动出去半个#

在 Chrome 控制台输入 $('.container').scrollTop=5，让 list 向上滚动 5px。

展示效果如下：

第三个步骤：列表开始滚动，第 1 个节点完全滚动出去#

在 Chrome 控制台输入 $('.container').scrollTop=10，让 list 向上滚动 10px。

展示效果如下：

此时，我们并不销毁第 1 个节点的 DOM，也不在列表最后为第 12 个节点创建新的 DOM，而是把第 1 个节点的 DOM 添加一个 transform(translateY(110px))，让他伪装成第 12 个节点的 DOM（注意，1 被重写为 12），紧跟在第 11 个节点的 DOM 后面。

代码如下：

<div class="container">  <div class="list">    <div class="item" style="transform: translateY(110px)">12</div>    <div class="item">2</div>    <div class="item">3</div>    <div class="item">4</div>    <div class="item">5</div>    <div class="item">6</div>    <div class="item">7</div>    <div class="item">8</div>    <div class="item">9</div>    <div class="item">10</div>    <div class="item">11</div>  </div></div>

展示效果如下：

在 Chrome 控制台输入 $('.container').scrollTop=18，让 list 向上滚动 18px（滚动 15px 有点看不清 12 这个字，所以这里多滚动一点）。
