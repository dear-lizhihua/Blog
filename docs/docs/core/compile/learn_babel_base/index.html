Babel 7 基础

重要声明#

本篇参考 Babel 详解 - 流云诸葛 一文重新梳理，主要用于自己学习，请勿转载。

什么是 Babel？Babel 如下能力#

一个 JavaScript compiler。

主要能力#

  • 转换语法。将 ECMAScript 2015+ 代码转换为 JavaScript 向后兼容版本的代码。

  • polyfill 目标环境中缺少的功能（通过 @babel/polyfill）

  • 源代码转换(codemods)。规范代码风格，可以直接将源码进行标准化转换。

其他能力#

  • Vue。转换 .vue 文件

  • React。转换 JSX 语法

  • 借助强类型语言的静态分析，检查 JavaScript 代码。例如 TypeScript 和 Flow

  • 使用 plugin 编写管道，以及使用 preset 使用插件集合。

  • 生成 sourcemap

  • 尽量遵循 ECMAScript 标准。在 presets 和 plugins 中，可能会看到 spec 选项，启用此选项，presets 和 plugins 就会按照更加符合规范的方式转换代码。为了加快转换速度，此选项一般不启用。

Babel 的能力来自 plugin#

全部能力都是通过 plugin 来完成的。每个 plugin 都有特定的处理代码的功能，只有将它配置到 babel 里面运行，才能对代码进行转换。

plugin 太多，就出现了 preset#

因为 plugin 太多，每次配置起来都特别麻烦，每个项目需要的 plugin，通常都是由类似的 plugin 组合而成，所以 Babel 推出了 presets。

每个 presets 都包含多个 plugin，只要配置了 presets，意味着这些 preset 包含的所有插件，都可能会参与到代码的转换中。

常见的 preset 有：preset-env, preset-state-0, preset-stage-2 等。

option 用于配置 plugin 和 preset#

plugin 和 preset 要经过配置才能生效，babel 自身也需要一些配置。

option 几种配置方式#

  1. （JSON 格式）package.json 文件，配置 babel 字段

  2. （JavaScript 格式）babel.config.js 文件

  3. （JSON 格式）.babelrc 文件

  4. （JavaScript 格式）.babelrc.js 文件

  5. Babel 命令行

  6. Babel API

Babel 集成 webpack 等构建工具#

Babel 可以和 webpack、Gulp 等构建工具结合使用

Babel 集成 ESLint 等 Lint 工具#

babel 还可以和 ESLint 等 Lint 工具结合使用。

Babel 将代码到 ES5 编译，用于在 Node.js 环境下执行#

在 Node.js 环境中，Babel 提供了 @babel/register，实现在 Node.js 加载 ES2015 文件时自动转换为 ES5 代码

Babel 补齐执行环境的功能，用于抹平差异#

一些新的 API 在 ES5 的运行环境中没有对应的实现，所以 Babel 的一项重要职责就是实现 polyfill 执行环境。

@babel/polyfill 的补齐能力，是使用 core-js/stable 和 regenerator-runtime/runtime 实现的。core-js 目前使用 v3 版本。regenerator-runtime 没有什么变化。需要掌握 core-js 是如何组织 ECMAScript 规范的各个模块实现的。

Babel 自动提取出多个输出文件中公共的辅助函数#

Babel 在转换过程中，会使用许多工具函数：helpers，例如 @babel/runtime/helpers/classCallCheck。

在不经过优化的时候，每个源文件的输出结果都会单独包含这些 helpers 代码。如果文件很多，就会导致大量的重复代码。所以 Babel 专门推出了 transform-runtime 来对这些 helpers 进行自动提取和其它优化。

什么是 ECMAScript#

Ecma International#

ECMA 是 “European Computer Manufacturers Association“ 的缩写，中文是欧洲计算机制造商协会。但是它拥有的标准牵涉到很多其他国家，因此组织决定改名 Ecma International 表明其国际性。

主要负责与有关组织合作开发通信技术和消费电子标准、鼓励准确的标准落实、和标准文件与相关技术报告的出版。

以下我们用 Ecma International 来表示这个机构。

ECMA 全部标准：https://www.ecma-international.org/publications-and-standards/standards/

ECMAScript#

ECMAScript 是一种脚本语言的标准化规范。是一种可以

  1. 在宿主环境中执行计算

  2. 并能操作可计算对象的

  3. 基于对象

的程序设计语言。

以下我们用 ECMAScript 来表示这个语言。

ECMA-262 标准#

ECMA International 机构将 JavaScript 标准化

  1. 将语言名称命名为 ECMAScript。我们通常称其为 JavaScript。

  2. 其标准编号名称为 ECMA-262。微软也把 C# 语言提交 ECMA International 进行标准化，编号 ECMA-334。

ECMA-262 标准内容：https://www.ecma-international.org/publications-and-standards/standards/ecma-262/

ECMA-334 标准内容：https://www.ecma-international.org/publications-and-standards/standards/ecma-334/

以下我们用 ECMA-262 标准 来表示这个标准。

ECMAScript 和 JavaScript 的关系#

标题里面的 JavaScript，是指当前在 browser、Node.js 等环境中运行的 JavaScript，不是早期提交给 ECMA International 标准化的 JavaScript。

前面提到，ECMAScript 是 ECMA-262 标准描述的编程语言名称。它其实是一个抽象语言名称。ECMA-262 标准内容，描述了这个语言该如何去实现，但是并没有给出特定运行环境的实现。

而在 browser、Node.js 等环境中运行的 JavaScript，是 ECMAScript 的一种实现。

ECMAScript 语言，允许实现对它的语言进行扩展，所以 JavaScript 会包含超出 ECMAScript 语言的功能。一份完整的 JavaScript，实际上包含三个主要部分：

  1. ECMAScript

  2. DOM

  3. BOM

以下我们用 JavaScript 来表示这个语言实现。

ECMA TC39#

ECMA TC39 是 Ecma International 里面专门负责 ECMA-262 也就是 ECMAScript 标准规范管理的一个分支机构。

ECMA TC39 目前管理 ECMA-262 标准的流程：The TC39 Process。

以下我们用 ECMA TC39 来表示这个标准管理机构。

ECMA-262 标准的修改流程#

任何人都可以向 ECMA TC39 提案，要求修改语言标准。

一种新的语法从提案到变成正式标准，需要经历五个阶段。每个阶段的变动都需要由ECMA TC39 批准。

  • Stage 0 - Strawman(稻草人)（草图阶段）

  • Stage 1 - Proposal(提案)（征求意见阶段）

  • Stage 2 - Draft(草案)（草案阶段）

  • Stage 3 - Candidate(候选)（候选人阶段）

  • Stage 4 - Finished(结束)（定案阶段）

从 The TC39 Process 可以看出，只有 stage 4 阶段的规范提案，才可能在当年 7 月份的时候，被加入到当年新标准发布中。

ECMA TC39 当前的所有提案都可以在 GitHub 上查看：https://github.com/tc39/ecma262

ECMAScript 发展历史#

版本号发布日期及标准文件连接版本名称
6https://262.ecma-international.org/6.0/ECMAScript 2015 (ES2015)，也称为 ES6
7https://262.ecma-international.org/7.0/ECMAScript 2016 (ES2016)，也称为 ES7
………………

现在通常所说的 ES6，其实是泛指 ES2015 发布以后，所有的 ES 版本，包括 ES6-ES10。说 ES8、9、10 的时候，说的其实就是 ES2017、ES2018、ES2019。

了解以上这些之后，对于一个前端开发者来说，关注 ECMA TC39 每年都有哪些新的规范提案，每年新的 ESMA-262 标准发布都添加了哪些特性，应该是一个保持关注技术新特性的方式。

TEST262#

Test262 GitHub：https://github.com/tc39/test262

这是官方提供的 ECMAScript 标准测试套件。

能通过 Test262 的用例，说明转码后的代码对 ECMA-262 标准的支持程度很高。

npm package 前面的 @符号#

从 Babel 7 起，package 都以 @babel 开头。这是 npm scope 定义的具有 scope 范围的 package

这类有 scope 的 package，只有 scope 的主体公司、机构和个人，才能往这个 scope 里面添加新的 package。也就是说以 @ 开头的 npm package，一定是官方推出或者官方认可，比较有权威性质。

  1. 普通 package，安装在 node_modules/{package} 这个文件夹下

  2. scope package，安装在 node_modules/@{scope}/{package} 这个文件夹下
