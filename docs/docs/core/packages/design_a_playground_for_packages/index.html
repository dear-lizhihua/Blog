设计一个 packages 的演示项目

设计意图#

monorepo 方式开发 packages 遇到的问题#

我们有时候会通过 monorepo 方式开发一些 package。

但是调试这些 package 通常来说比较麻烦，如果按照常规的流程：

  1. 我们需要在 packages 下，把开发代码构建为生成代码，然后发布一个测试版本 1.0.0-alpha 到线上。
  2. 我们需要在 project 下，把刚才发布的 1.0.0-alpha 安装上，再重新运行项目查看效果。

这样的开发流程要经过构建、发布、安装、重新运行四步，操作起来也很繁琐。

开发 packages 的正确方式：使用 yarn workspaces，建立示例代码和框架代码之间的关联#

有没有一种办法，简化掉发布和安装两个流程，直接在开发代码构建后，再重新运行下项目，就可以看到效果。

方法是有的。

使用 yarn workspaces 软链接的方式，把 examples 目录和 packages 目录中的 package 都平铺地暴露在根路径 node_modules 中，使得两个 package 之间可以直接引用。

node_modules├── @examples/project└── @packages/framework

这样在 examples/project 中，引用 @packages/framework 模块，就会去 node_nodules/@packages/framework 找到此源码目录下的 package.json，再通过 module key 找到 dist/index.esm.js。

我们每次修改 @packages/framework 源码，手动或自动执行一次构建，把生成代码重新构建到 dist/index.esm.js 中，然后 examples/project 就可以使用新的生成代码来运行。

展示代码#

所以，我们需要创建一个 playground 演示项目，来帮助我们集成和调试一个 package。

项目目录结构#

假想我们的项目目录结构如下：

playground├── examples│   └── project # 通过此示例项目，来调试 package│       ├── src│       │   ├── views│       │   │   └── pages│       │   │       └── home│       │   │           └── index.vue│       │   └── main.js│       ├── index.html│       └── package.json├── packages│   └── framework # 此目录是我们要调试的 package│       ├── dist│       │   ├── index.esm.js│       │   └── style.css│       ├── src│       │   ├── styles│       │   │   └── index.scss│       │   └── index.js│       └── package.json└── package.json

playground 项目根目录下的 package.json#

{  "name": "playground",  "private": true,  "workspaces": [ // 通过 yarn workspaces 暴露两个 package 到 node_modules 中。    "examples/*",    "packages/*"  ]}

packages/framework 目录下的 package.json#

{  "name": "@packages/framework", // 这里是 package 名称  "version": "1.0.0",  "module": "dist/index.esm.js", // 将 @packages/framework 模块指向到 dist/index.esm.js 文件}

examples/project 目录下的 package.json#

{  "name": "@examples/project", // 这里是 package 名称  "dependencies": {    "@packages/framework": "^1.0.0" // 直接引用 @packages/framework 模块  },}
