理解前端路由 - 什么是 History API？

准备阶段#

在全局环境安装 serve#

为了更好地演示，我们需要通过 npm，在全局环境安装 serve。

$ npm install -g serve

创建一个 index.html#

在一个 example 目录下

$ mkdir example$ cd example$ vi index.html
# 输入如下内容后保存<!DOCTYPE html><html><head>  <meta charset='UTF-8'>  <title>History API</title></head><body>  <!-- 这里是一些示例代码 --></body></html>

以 spa 方式运行 index.html#

$ serve -s

此时就以 spa 方式，打开了一个 HTML 页面。

创建一个初始的页面#

不要在浏览器输入 URL 创建初始页面，否则 history 栈中会多一条记录。

我们通过 window.open('http://localhost:5000') 创建一个初始的页面。

此时 history 栈的示意图如下：

init <--

并且前进和后退按钮都不可点击，这是我们想要的初始页面。

浏览器提供的属性#

什么是 history？#

history 是个栈结构，我们可以将其理解为一个栈，可以对其进行 push, pop 来添加和删除记录。

现在，在控制台打印 history。

console.log(history)
{  length: 1  scrollRestoration: "auto"  state: null  __proto__: History}

可以看出：history 实例是一个栈结构，有 length 属性，并且其通过 __proto__ 继承 History 实例对象。

什么是 History 和 History.prototype？#

History 是一个构造函数。

console.log(History)
ƒ History() { [native code] }

History.prototype 是一个原型对象。

console.log(History.prototype)
{  back: ƒ back()  forward: ƒ forward()  go: ƒ go()  length: (...)  pushState: ƒ pushState()  replaceState: ƒ replaceState()  scrollRestoration: (...)  state: (...)  constructor: ƒ History()  Symbol(Symbol.toStringTag): "History"  get length: ƒ length()  get scrollRestoration: ƒ scrollRestoration()  set scrollRestoration: ƒ scrollRestoration()  get state: ƒ state()  __proto__: Object}

history 对象和 History 实例对象是通过 __proto__ 实现的的继承关系#

new History() 创建出来的实例对象，会被 history 对象 通过 __proto__ 继承过去。

因此 history 可以直接调用 History.prototype 上面的 go(), back() 方法。

浏览器提供的不修改栈的方法#

注意 history.go([delta]), history.forward(), history.back() 这些方法

  1. 都是异步函数
  2. 都 触发 popstate 事件。

注意这里都是异步函数，不要直接连续调用，避免出现意想不到的结果。

history.go([delta])#

不修改栈，相对于当前在 history 栈中的位置，向前或向后移动。

history.forward()#

等于 history.go(1)。

不修改栈，而是将栈指针往后指向一位。

history.back()#

等于 history.go(-1)。

不修改栈，而是将栈指针往前指向一位。

浏览器提供的修改栈的方法#

注意 history.pushState(), history.replaceState() 这些方法

  1. 都是异步函数
  2. 都 不触发 popstate 事件。

注意这里都是异步函数，不要直接连续调用，避免出现意想不到的结果。

无法实现出栈#

我们在上面了解到 history 是一个栈。

栈结构是具有 push 入栈 和 pop 出栈 能力的。

但是浏览器设计的 history 栈，允许 push 入栈，但 不允许 pop 出栈，仅额外增加了一个 replace 替换指针指向的栈记录 的能力。

history.pushState()#

我们调用 history.pushState 推入一条记录，再次控制台打印 history。

history.pushState({page: 1}, 'title 1', '/page1')console.log(history)
{  length: 2  scrollRestoration: "auto"  state: {page: 1}  __proto__: History}

可以看出，栈的长度变为 2，而且此时的 state 是我们刚才调用 pushState 传递的第一个参数的值 {page: 1}。

此时 history 栈的示意图如下：

/page1 <--init

我们继续做如下操作：

  • 调用 history.back() 回到 init 记录，打印 state，得到 null。
  • 调用 history.forward() 回到 /page1 这条记录，打印 state，得到 {page: 1}。
  • 刷新页面，打印 state，得到 {page: 1}。

也就是说，state 是跟着 history 栈存储的，刷新页面也不会丢失。

history.replaceState()#

<!DOCTYPE html><html>  <head>    <meta charset="UTF-8">    <title>History API</title>  </head>  <body>    <button onclick="stackPush()">stack push</button>    <button onclick="stackBack()">stack back</button>    <button onclick="stackReplace()">stack replace</button>    <script>      window.stackPush = function () {        history.pushState({page: 1}, 'title 1', '/page1')        history.pushState({page: 2}, 'title 2', '/page2')      }      window.stackBack = function () {        history.back()      }      window.stackReplace = function () {         history.replaceState({page: 3}, 'title 3', '/page3')      }    </script>  </body></html>

点击 stack push 按钮后，history 栈的示意图如下：

/page2 <--/page1init

点击 stack back 按钮和 stack replace 按钮后，history 栈的示意图如下：

/page2/page3 <--init

这说明 history.replaceState() 是替换当前指针指向的记录，而不影响其他的记录。

History API 提供的方法都不会刷新页面#

注意以下：

  1. History API 提供的方法，只是修改栈和操作指针，都不会刷新页面。
  2. location.href 赋值，既 push 入栈，也刷新页面。

所以使用 Vue, React 等框架时，要设计好 history 栈的交互，慎用 location.href 跳转页面，避免造成页面刷新。

浏览器提供的事件#

popstate 事件#

浏览器只提供一个 popstate 事件。可以使用两种方式监听此事件

window.addEventListener('popstate', (event) => {  console.log(event)}, false)// 或者window.onpopstate = (event) => {  console.log(event)}

建议使用 window.addEventListener('popstate', function(event) {}, false) 方式。

谁触发 popstate 事件？#

<!DOCTYPE html><html>  <head>    <meta charset="UTF-8">    <title>History API</title>  </head>  <body>    <button onclick="stackTake()">stack take</button>    <button onclick="stackBack()">stack back</button>    <button onclick="stackForward()">stack forward</button>    <button onclick="stackGo()">stack go</button>    <button onclick="stackPush()">stack push</button>    <button onclick="stackReplace()">stack replace</button>    <script>      // 先监听 popstate 事件      window.addEventListener('popstate', (event) => {        console.log(event)      }, false)
      // 再向 history 栈中添加三条记录，并且指针指向到栈顶      window.stackTake = function () {        history.pushState({page: 1}, 'title 1', '/page1')        history.pushState({page: 2}, 'title 2', '/page2')        history.pushState({page: 3}, 'title 3', '/page3')      }      window.stackBack = function () {        history.back()      }      window.stackForward = function () {        history.forward()      }      window.stackGo = function () {        history.go(-2)      }      window.stackPush = function () {        history.pushState({page: 4}, 'title 4', '/page4')      }      window.stackReplace = function () {        history.replaceState({page: 5}, 'title 5', '/page5')      }    </script>  </body></html>

通过逐个点击上面的按钮，我们可以观察到：

  1. 所有修改 history 栈的方法、不修改指针的方法，都不会触发 popstate 事件。例如 调用 history.pushState() 或 history.replaceState() 都不会触发 popstate 事件。
  2. 所有不修改 history 栈、修改指针的方法，都会触发 popstate 事件。真正触发 popstate 事件的是 history.back(), history.forward(), history.go()。

也就是说，只有 用户在浏览器点击前进和后退按钮 或者 JavaScript 代码中调用 history.back(), history.forward(), history.go() 这两种情况会触发 popstate 事件。
