理解前端部署环境

区分部署环境#

在项目部署时，通常会分为三种环境：开发环境、测试环境、线上环境。

开发环境(development)#

通常是在本地运行起来的开发服务。

访问链接类似 http://localhost:<port>/ 或 http://127.0.0.1:<port>/。

测试环境(test)#

测试环境是用于 QA 检查是否存在流程上、逻辑上的问题。

访问链接类似 https://test-account.domain.com/。

生产环境(production)#

生产环境是交付使用的正式环境，一定要经过 QA 环节进行充分验证，才能部署代码。

访问链接类似 https://account.domain.com/。

检查部署环境#

我们以 https://test-account.domain.com/ 和 https://account.domain.com/ 为例。

解析 URL 对象#

我们先通过 url 将 location.href 解析为 URL 对象。

import url from 'url'const parsedUrl = url.parse(location.href)

我们得到如下解析结果：

{  "protocol": "https:",  "slashes": true,  "auth": null,  "host": "test-account.domain.com",  "port": null,  "hostname": "test-account.domain.com",  "hash": null,  "search": null,  "query": null,  "pathname": "/",  "path": "/",  "href": "https://test-account.domain.com/"}

通过 hostname 来检查域名#

parsedUrl.hostname // "test-account.domain.com"

我们通常认为：

  1. 非 test- 开头的域名是生产环境域名。
  2. test- 开头的域名是测试环境域名。

也可以有其他方式来区分生产环境域名和开发环境域名。

检查生产环境域名#

const isProd = value => /^(?!test-)(.+)\.domain\.com/g.test(value)console.log(isProd('localhost')) // falseconsole.log(isProd('127.0.0.1')) // falseconsole.log(isProd('test-account.domain.com')) // falseconsole.log(isProd('account.domain.com')) // true

检查开发环境域名#

const isTest = value => /^test-(.+)\.domain\.com/g.test(value)console.log(isTest('localhost')) // falseconsole.log(isTest('127.0.0.1')) // falseconsole.log(isTest('test-account.domain.com')) // trueconsole.log(isTest('account.domain.com')) // false

获取项目名称#

const isProd = value => /^(?!test-)(.+)\.domain\.com/g.test(value)const isTest = value => /^test-(.+)\.domain\.com/g.test(value)const getProject = (value) => {  if (isProd(value)) {    return /^(?!test-)(.+)\.domain\.com/g.exec(value)[1]  } else if (isTest(value)) {    return /^test-(.+)\.domain\.com/g.exec(value)[1]  } else {    return ''  }}
const matched1 = getProject('localhost')const matched2 = getProject('127.0.0.1')const matched3 = getProject('test-account.domain.com')const matched4 = getProject('account.domain.com')console.log(matched1 === '') // trueconsole.log(matched2 === '') // trueconsole.log(matched3) // 'account'console.log(matched4) // 'account'
