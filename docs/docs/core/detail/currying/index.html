柯里化实现 add(1)(2)(3)

思路#

我们来分析问题。

首先，add 是个函数#

function add(a) {}

其次，add 函数执行后返回的也必须是个函数#

我们假想为这个命名 addReturnFn，这样才能执行 addReturnFn(2)

function add(a) {  return function addReturnFn(b) {  }}

最后，addReturnFn 函数执行后返回的也必须是个函数#

我们假想为这个命名 addReturnFnReturnFn，这样才能执行 addReturnFnReturnFn(3)

function add(a) {  return function addReturnFn(b) {    return function addReturnFnReturnFn(c) {      return a + b + c    }  }}

用一行代码实现#

ECMAScript 2015 提供箭头函数

const add = a => b => c => a + b + c

柯里化#

定义#

在计算机科学中，柯里化（英语：currying），是把接收多个参数的函数变换成接受一个单一参数的函数，并且返回接受余下的参数而且返回结果的新函数的技术。

题目发生变化#

概念有点难懂，用代码解释下。

如果我们现在要求，给定一个函数 function a(a, b, c) { return a + b + c }，它可以接收三个参数 a, b, c，请将此函数柯里化，使其满足

  1. add(1)(2)(3) => 6
  2. add(1, 2)(3) => 6
  3. add(1)(2, 3) => 6
  4. add(1, 2, 3) => 6

我的分析思路#

首先，通过外部的 fn.length 计算函数接收的形式参数个数 n，此题中是 3。通过内部的 (...args) 将实际传入的参数转为数组，再对其计算数组长度。

其次，我们每消费几个参数，就会剩余 (n - 几) 个参数。 例如，在上面第一个示例中，第一次消费 1 个参数，那剩余 2 个参数；第二次消费 1 个参数，那剩余 1 个参数；第三次消费 1 个参数，此时参数个数为 0。

最后，当参数全部消费完毕，就执行最终计算。

我的代码实现#

以下代码完全按照自己以上所述的思路实现。

function a(a, b, c) {  return a + b + c}function curring(fn) {  const fnLength = fn.length // 第一次包装函数 fn 时，可以通过 fn.length 得到函数 fn 的参数个数  let _args = [] // 这里我们收集陆陆续续接收到的参数
  // 这里我们为此匿名函数命名 curringA，此函数承担着两个任务：  // 1. 担任收集实际参数的任务。当收集的参数数量不够时，继续进行收集。  // 2. 担任执行函数计算的任务。当收集参数任务完成后，此函数需要真正执行。  return function curringA(...args) {    _args = _args.concat(args) // 通过外层函数作用域拿到之前收集到的参数，和新得到的参数放在一起    if (_args.length >= fnLength) { // 这里注意，收集到的全部参数在数量上要多余实际要用到的参数数量，这时才最终执行。      const res = fn(..._args)      _args = [] // 这里别忘了销毁之间收集的参数，在下一次调用之前保持内部状态干净      return res    } else {      return curringA // 参数不够，继续递归收集参数    }  }}const curringA = curring(a); // 使用函数 curring 对函数 a 进行包装，得到柯里化后的新函数 curringAconsole.log(curringA(1)(2)(3)); // 6console.log(curringA(1, 2)(3)); // 6console.log(curringA(1)(2, 3)); // 6console.log(curringA(1, 2, 3)); // 6
